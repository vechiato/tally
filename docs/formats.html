settings.yaml

The main configuration file. Created by tally init.

year: 2025

currency_format: "${amount}"  # Optional: defaults to $

merchants_file: config/merchants.rules
views_file: config/views.rules    # Optional: custom report sections

data_sources:
  - name: Chase
    file: data/chase.csv
    format: "{date:%m/%d/%Y},{description},{amount}"

  - name: Bank of America
    file: data/bofa.csv
    format: "{date:%m/%d/%Y},{description},{amount}"

Data Sources

Each data source represents a CSV file from a bank or credit card.

Field Required Description
name Yes Display name (e.g., "Chase", "AMEX Gold")
file Yes Path to CSV file (relative or absolute)
format Yes Format string describing CSV columns
decimal_separator No For European format: "," (1.234,56)
columns No Custom column mappings (see Format Strings)
delimiter No Column delimiter: tab, whitespace, or regex:pattern
has_header No Set to false if file has no header row

Non-CSV Files

For tab-separated, whitespace-delimited, or fixed-width formats, use the delimiter option:

# Tab-separated file
- name: Bank Statement
  file: data/stmt.tsv
  delimiter: tab
  format: "{date:%m/%d/%Y},{description},{amount}"

# Fixed-width with regex capture groups
- name: BOA Statement
  file: data/stmt.txt
  delimiter: "regex:^(\\d{2}/\\d{2}/\\d{4})\\s+(.+?)\\s+([-\\d,]+\\.\\d{2})\\s+([\\d,]+\\.\\d{2})$"
  format: "{date:%m/%d/%Y},{description},{-amount},{_}"
  has_header: false

Regex delimiters use capture groups () to extract columns. Each group becomes a column matched by the format string.

Format Strings

Format strings describe how to parse each CSV column. They map your CSV structure to the fields Tally needs.

Basic Tokens

Token Description
{date:%m/%d/%Y} Date with format specifier
{description} Transaction description
{amount} Amount column (keep sign as-is)
{-amount} Amount column (negate the value)
{+amount} Amount column (take absolute value)
{_} Skip this column
{custom_name} Capture column as custom field
{location} Location code for travel detection

Amount Signs

Different banks format amounts differently. Use the right token:

# Charges are positive, payments are negative (most credit cards)
format: "{date},{description},{amount}"

# Charges are negative, deposits are positive (most bank accounts)
# Use {-amount} to flip the sign so spending is positive
format: "{date},{description},{-amount}"

# All amounts are positive, separate debit/credit columns
# Use {+amount} to ensure positive, skip the other column
format: "{date},{description},{+amount},{_}"

Capturing Custom Fields

Any token besides the reserved ones becomes a custom field you can use in rules:

# CSV: Date, Type, Memo, Vendor, Amount
format: "{date:%m/%d/%Y},{txn_type},{memo},{vendor},{amount}"

# Now use in merchants.rules:
# match: field.txn_type == "WIRE"
# match: contains(field.memo, "Invoice")

Combining Columns into Description

Some CSVs split transaction info across multiple columns. Use columns.description to combine them:

# CSV: Date, Category, Payee, Reference, Amount
- name: My Bank
  file: data/bank.csv
  format: "{date:%m/%d/%Y},{category},{payee},{ref},{amount}"
  columns:
    description: "{payee} - {category}"
# Result: "AMAZON - Shopping" instead of just one column

The template can reference any captured field:

# Include reference number for matching
columns:
  description: "{payee} ({ref})"
# Result: "AMAZON (INV-12345)"

Examples

# Simple: date, description, amount
format: "{date:%m/%d/%Y},{description},{amount}"

# Skip columns you don't need
format: "{date:%m/%d/%Y},{_},{description},{_},{amount}"

# Bank account with negated amounts
format: "{date:%m/%d/%Y},{description},{-amount}"

# European bank with custom description
- name: German Bank
  file: data/german.csv
  format: "{date:%d.%m.%Y},{_},{txn_type},{vendor},{_},{amount}"
  decimal_separator: ","
  columns:
    description: "{vendor} ({txn_type})"

Date Formats

Common date format specifiers:

Format Example Notes
%m/%d/%Y 01/15/2025 US format
%d/%m/%Y 15/01/2025 European format
%Y-%m-%d 2025-01-15 ISO format
%d.%m.%Y 15.01.2025 German format
%m-%d-%Y 01-15-2025 US with dashes

Currency Format

Display amounts in your local currency:

# US Dollars (default)
currency_format: "${amount}"      # → $1,234.56

# Euro
currency_format: "€{amount}"      # → €1,234.56

# British Pound
currency_format: "£{amount}"      # → £1,234.56

# Polish Zloty (suffix)
currency_format: "{amount} zł"   # → 1,234.56 zł

# Japanese Yen (no decimals)
currency_format: "¥{amount}"      # → ¥1,234

European Number Format

For CSVs using comma as decimal separator:

- name: German Bank
  file: data/german.csv
  format: "{date:%d.%m.%Y},{description},{amount}"
  decimal_separator: ","  # Parses 1.234,56 correctly

Travel Detection

Tally can automatically detect travel-related transactions using location data from your CSVs.

Capturing Location

If your bank includes a state/country code in the CSV, capture it with {location}:

# CSV: Date, Description, Amount, State
format: "{date:%m/%d/%Y},{description},{amount},{location}"

Defining Home Locations

Transactions outside these locations are classified as travel:

home_locations:
  - WA
  - OR
  - CA

Custom Travel Labels

Map location codes to readable names in reports:

travel_labels:
  NY: New York Trip
  TX: Austin Conference
  GB: London Vacation

CLI Commands

Command Description
tally init [dir] Create a new budget folder
tally workflow Show next steps (context-aware)
tally run Generate HTML spending report
tally inspect <csv> Analyze CSV structure
tally discover Find uncategorized transactions
tally explain <merchant> Explain merchant classification
tally diag Debug configuration issues
tally reference Show full reference docs

Output Formats

# Different output formats
tally run                      # HTML report (default)
tally run --format json        # JSON with reasoning
tally run --format markdown    # Markdown report
tally run --format summary     # Text summary only

# Verbosity levels
tally run -v                   # Include decision trace
tally run -vv                  # Include thresholds, CV values

Filtering

# Filter by classification
tally run --format json --only monthly,variable

# Filter by category
tally run --format json --category Food

# Explain specific classifications
tally explain --classification monthly
tally explain --category Subscriptions
tally explain --tags business
